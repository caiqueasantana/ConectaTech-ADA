Desenvolvimento do Módulo Embarcado

O Módulo de Hardware Embarcado é o coração físico do Kit ConectaTech ADA, permitindo que os conceitos de programação se materializem em ações no mundo real. Nesta seção, detalhamos o hardware selecionado, o firmware desenvolvido e os desafios enfrentados no processo.

Hardware Utilizado

A escolha dos componentes foi guiada pelo critério de baixo custo, disponibilidade, facilidade de uso para iniciantes e capacidades adequadas para os módulos didáticos. O principal microcontrolador utilizado é o ESP32, conhecido por sua conectividade Wi-Fi e Bluetooth integradas, além de uma boa quantidade de GPIOs (General Purpose Input/Output).

  • Microcontrolador: ESP32 DevKit V1 (ou similar)
    • **Justificativa:** Excelente custo-benefício, Wi-Fi e Bluetooth integrados, poder de processamento para as tarefas propostas, vasta comunidade e documentação. Sua arquitetura demonstra princípios de **Arquitetura e Organização de Computadores** que são discutidos nos módulos.
  • Sensores:
    • **Sensor de Luminosidade (LDR):** Para demonstrações de leitura analógica e controle baseado na luz ambiente.
    • **Sensor de Temperatura e Umidade (DHT11/22 - Opcional):** Para coleta de dados ambientais mais complexa.
  • Atuadores:
    • **LEDs (diferentes cores):** Para demonstrações de saída digital e controle visual.
    • **Resistores:** Componentes de proteção essenciais.
    • **Protoboard e Jumpers:** Para montagem flexível dos circuitos.

Firmware (Código)

O firmware é o software que roda diretamente no microcontrolador ESP32, escrito principalmente em C++ utilizando o ambiente Arduino IDE. Ele é responsável por controlar os componentes, ler os sensores e gerenciar a comunicação com o módulo de software multiplataforma.

Estrutura e Lógica Principal

O firmware é estruturado para ser modular, facilitando a compreensão e a expansão. As principais lógicas implementadas são:

  • Inicialização (setup()): Configuração de pinos, inicialização da comunicação serial/Wi-Fi.
  • Loop Principal (loop()): Leitura contínua de sensores, processamento de dados, controle de atuadores e comunicação de dados. Aplicação de **Algoritmos e Lógica de Programação** para tomada de decisões.
  • Funções de Comunicação: Rotinas para receber comandos da plataforma multiplataforma e enviar dados dos sensores.

Exemplo de Código: Leitura de Sensor e Controle Básico

Abaixo, um trecho de código que demonstra a leitura de um sensor de luminosidade (LDR) e o controle de um LED conectado ao ESP32, baseado no valor lido. Este exemplo ilustra a interconexão do hardware com a lógica programada.


// Código para ESP32: Ler LDR e Controlar LED, e enviar via Serial/Wi-Fi
#include <Arduino.h>
#include <WiFi.h> // Se estiver usando comunicação Wi-Fi

const int LED_PIN = 2;       // GPIO para o LED
const int LDR_PIN = 34;      // GPIO para o LDR (ADC1_CH6)

// Configurações de Wi-Fi (se aplicável para comunicação TCP/UDP/MQTT)
// const char* ssid = "SEU_WIFI";
// const char* password = "SUA_SENHA_WIFI";

void setup() {
  Serial.begin(115200);
  pinMode(LED_PIN, OUTPUT);
  // pinMode(LDR_PIN, INPUT); // LDR_PIN já é input por padrão para analogRead

  // Conexão Wi-Fi (se usar)
  // WiFi.begin(ssid, password);
  // while (WiFi.status() != WL_CONNECTED) {
  //   delay(500);
  //   Serial.println("Conectando ao WiFi...");
  // }
  // Serial.println("WiFi Conectado!");
  // Serial.print("Endereço IP: ");
  // Serial.println(WiFi.localIP());
}

void loop() {
  int valorLDR = analogRead(LDR_PIN); // Lê o valor do LDR (0-4095 para ESP32)
  
  Serial.print("Luminosidade: ");
  Serial.println(valorLDR); // Envia o valor via serial para o PC ou gateway

  // Exemplo de lógica: Acende o LED se estiver escuro (valor LDR baixo)
  if (valorLDR < 1500) { // Valor de threshold, ajuste conforme necessário
    digitalWrite(LED_PIN, HIGH); // Acende o LED
    // Serial.println("LED LIGADO (Ambiente escuro)");
  } else {
    digitalWrite(LED_PIN, LOW);  // Apaga o LED
    // Serial.println("LED DESLIGADO (Ambiente claro)");
  }

  // Simula recebimento de comando da plataforma multiplataforma (via Serial)
  if (Serial.available() > 0) {
    String command = Serial.readStringUntil('\n');
    command.trim(); // Remove espaços em branco
    if (command == "LIGAR_LED") {
      digitalWrite(LED_PIN, HIGH);
      Serial.println("Comando: LED LIGADO");
    } else if (command == "DESLIGAR_LED") {
      digitalWrite(LED_PIN, LOW);
      Serial.println("Comando: LED DESLIGADO");
    }
  }

  delay(1000); // Espera 1 segundo antes da próxima leitura/execução
}

Desafios e Soluções

O desenvolvimento embarcado trouxe desafios únicos, nos quais aplicamos diretamente os conhecimentos das disciplinas da FATEC:

  • Gerenciamento de Recursos: Microcontroladores possuem memória e processamento limitados. Otimizamos o código e o uso de bibliotecas, aplicando princípios de **Arquitetura e Organização de Computadores** para garantir a eficiência.
  • Estabilidade da Comunicação: A comunicação serial e Wi-Fi entre o embarcado e a plataforma multiplataforma exigiu atenção à sincronização e tratamento de erros. A compreensão de como os **Sistemas Operacionais** subjacentes gerenciam as interrupções e as tarefas foi fundamental.
  • Depuração de Hardware: A identificação de falhas no circuito ou na programação exigiu métodos sistemáticos de depuração, utilizando ferramentas como o monitor serial e multímetro.

A superação desses desafios foi essencial para a robustez do Kit ConectaTech ADA e para o aprendizado prático em desenvolvimento de sistemas embarcados.
