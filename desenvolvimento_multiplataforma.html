Desenvolvimento do Módulo Multiplataforma

O Módulo de Software Multiplataforma é a interface amigável do Kit ConectaTech ADA, permitindo que os usuários (alunos e comunidade) interajam de forma intuitiva com a lógica de programação e controlem o módulo embarcado. Esta seção detalha as tecnologias utilizadas, a arquitetura do software e as funcionalidades implementadas.

Tecnologias Utilizadas

Para garantir acessibilidade e facilidade de aprendizado, a escolha das tecnologias focou em soluções que permitissem o desenvolvimento multiplataforma com um bom balanço entre simplicidade e capacidade. Optamos por:

  • Linguagem de Programação Principal: Python
    • **Justificativa:** Sua sintaxe clara e legibilidade a tornam ideal para iniciantes em programação. Possui bibliotecas robustas para interface gráfica (Tkinter) e comunicação serial/rede.
  • Framework/Biblioteca de Interface Gráfica: Tkinter (para aplicação desktop)
    • **Justificativa:** Integrado ao Python, não exige instalação adicional complexa, facilitando o uso em diversos computadores da FATEC e da comunidade. É multiplataforma por natureza (Windows, macOS, Linux).
  • Comunicação com o Embarcado: PySerial (para comunicação serial via USB) ou WebSockets/MQTT (para comunicação Wi-Fi)
    • **Justificativa:** Fornecem métodos eficientes e confiáveis para o software interagir com o microcontrolador ESP32.
  • Estruturas de Dados: Listas, Dicionários e Filas em Python para gerenciamento de comandos, logs e dados de sensores.

Arquitetura do Software e UI/UX

A arquitetura do módulo multiplataforma foi desenhada para ser modular e extensível, com foco na usabilidade (UX) e em uma interface visual clara (UI) para o público iniciante.

  • Arquitetura de Camadas: O software é dividido em camadas (Interface de Usuário, Lógica de Negócio/Processamento, Camada de Comunicação com Hardware) para melhor organização e manutenção.
  • Interface de Usuário (UI):
    • **Ambiente de Programação Visual:** Um editor de texto simplificado ou um sistema de "blocos" (se implementado) onde o usuário escreve/monta seus **Algoritmos**.
    • **Painel de Controle:** Botões para enviar comandos (ex: ligar/desligar LED) e áreas para exibir dados recebidos do hardware (ex: valor de luminosidade).
    • **Logs de Comunicação:** Uma área para exibir a troca de mensagens entre o software e o hardware, facilitando a depuração e o entendimento.
  • Experiência do Usuário (UX): Priorizamos a simplicidade, com feedback visual claro (ex: botão muda de cor ao clicar), mensagens de erro amigáveis e um fluxo de trabalho intuitivo para cada desafio didático.

Código (Exemplos de Implementação)

Apresentamos abaixo trechos de código-fonte que ilustram a implementação das principais funcionalidades do módulo multiplataforma, demonstrando a aplicação de **Algoritmos e Lógica de Programação**, **Estruturas de Dados** e comunicação com o hardware.

Exemplo de Código: Interface e Comunicação (Python + Tkinter + PySerial)

Este snippet demonstra a criação de uma janela simples com um botão para controlar um LED no módulo embarcado e um label para exibir dados de sensores via comunicação serial.


# Exemplo de Módulo Multiplataforma (Python + Tkinter + PySerial)
import tkinter as tk
from tkinter import ttk
import serial.tools.list_ports # Para listar portas seriais disponíveis

# --- Configuração da Comunicação Serial ---
# Defina a porta serial e baud rate (deve corresponder ao ESP32)
SERIAL_PORT = 'COM3' # Altere para a porta correta do seu ESP32 (ex: '/dev/ttyUSB0' no Linux)
BAUD_RATE = 115200

ser = None # Objeto serial global

def connect_serial():
    global ser
    try:
        ser = serial.Serial(SERIAL_PORT, BAUD_RATE, timeout=1)
        status_label.config(text=f"Status: Conectado a {SERIAL_PORT}", foreground="green")
        read_serial_data() # Inicia a leitura contínua
    except serial.SerialException as e:
        status_label.config(text=f"Status: Erro de Conexão ({e})", foreground="red")
        print(f"Erro ao conectar: {e}")

def disconnect_serial():
    global ser
    if ser and ser.is_open:
        ser.close()
        status_label.config(text="Status: Desconectado", foreground="orange")
        print("Porta serial desconectada.")

def send_command_to_esp32(command):
    if ser and ser.is_open:
        ser.write(command.encode('utf-8'))
        print(f"Comando enviado: {command.strip()}")
    else:
        print("Erro: Porta serial não conectada.")

def toggle_led_on_esp32():
    if led_button["text"] == "Ligar LED":
        send_command_to_esp32("LIGAR_LED\n")
        led_button.config(text="Desligar LED", style="Accent.TButton")
    else:
        send_command_to_esp32("DESLIGAR_LED\n")
        led_button.config(text="Ligar LED", style="TButton")

def read_serial_data():
    global ser
    if ser and ser.is_open and ser.in_waiting > 0:
        try:
            line = ser.readline().decode('utf-8').strip()
            if line:
                # Exemplo de tratamento de dados: "Luminosidade: 850"
                if "Luminosidade:" in line:
                    luminosity_value = line.split(":")[1].strip()
                    luminosity_label.config(text=f"Luminosidade: {luminosity_value}")
                # Adicione mais parsers para outros tipos de dados ou comandos do ESP32
                print(f"Recebido: {line}")
            root.after(100, read_serial_data) # Chama a função novamente
        except Exception as e:
            print(f"Erro na leitura serial: {e}")
            root.after(100, read_serial_data)
    else:
        root.after(100, read_serial_data) # Continua tentando ler mesmo se não tiver dados

# --- Configuração da Interface Gráfica (Tkinter) ---
root = tk.Tk()
root.title("ConectaTech ADA - Controle")
root.geometry("400x300")
root.resizable(False, False)

# Estilos (para deixar mais bonito)
style = ttk.Style()
style.theme_use('clam') # 'clam' ou 'alt' ou 'default'
style.configure("TButton", padding=6, relief="flat", background=var(--secondary-color), foreground=var(--text-light))
style.map("TButton",
          background=[('active', var(--secondary-color))],
          foreground=[('active', var(--text-light))])

style.configure("Accent.TButton", background=var(--accent-color), foreground=var(--text-light))
style.map("Accent.TButton",
          background=[('active', '#218838')])


main_frame = ttk.Frame(root, padding="20")
main_frame.pack(fill=tk.BOTH, expand=True)

# Controles de Conexão
conn_frame = ttk.LabelFrame(main_frame, text="Conexão Serial", padding="10")
conn_frame.pack(pady=10, fill=tk.X)

port_label = ttk.Label(conn_frame, text="Porta:")
port_label.grid(row=0, column=0, padx=5, pady=5)

# Lista de portas disponíveis (dinâmica)
ports = [p.device for p in serial.tools.list_ports.comports()]
selected_port = tk.StringVar(root)
if ports:
    selected_port.set(ports[0]) # Seleciona a primeira porta por padrão
    SERIAL_PORT = ports[0]
else:
    selected_port.set("Nenhuma porta encontrada")

port_dropdown = ttk.OptionMenu(conn_frame, selected_port, *ports, command=lambda port: globals().update(SERIAL_PORT=port))
port_dropdown.grid(row=0, column=1, padx=5, pady=5, sticky=tk.W+tk.E)

connect_button = ttk.Button(conn_frame, text="Conectar", command=connect_serial)
connect_button.grid(row=0, column=2, padx=5, pady=5)

disconnect_button = ttk.Button(conn_frame, text="Desconectar", command=disconnect_serial)
disconnect_button.grid(row=0, column=3, padx=5, pady=5)

status_label = ttk.Label(conn_frame, text="Status: Desconectado", foreground="orange")
status_label.grid(row=1, column=0, columnspan=4, pady=5)


# Controles do Kit
kit_frame = ttk.LabelFrame(main_frame, text="Controle do Kit", padding="10")
kit_frame.pack(pady=10, fill=tk.X)

led_button = ttk.Button(kit_frame, text="Ligar LED", command=toggle_led_on_esp32)
led_button.pack(pady=10)

luminosity_label = ttk.Label(kit_frame, text="Luminosidade: N/A")
luminosity_label.pack(pady=10)

root.mainloop()

# Fecha a porta serial ao sair
if ser and ser.is_open:
    ser.close()

Este código ilustra como a interface gráfica (Tkinter) se comunica com o hardware embarcado via serial, enviando comandos e exibindo dados. Ele exemplifica a aplicação de **Algoritmos e Lógica de Programação** para o controle e a utilização de **Estruturas de Dados** implícitas no manuseio das informações. A escolha do Python e Tkinter garante a capacidade **multiplataforma** para desktop.

Desafios e Soluções

O desenvolvimento multiplataforma apresentou desafios importantes que exigiram a aplicação de conhecimentos de **Sistemas Operacionais** e de design de software:

  • Compatibilidade entre Sistemas Operacionais: Garantir que a aplicação Python com Tkinter funcionasse igualmente bem em Windows, macOS e Linux, lidando com diferenças de caminhos de portas seriais e gerenciamento de janelas.
  • Comunicação Assíncrona: Manter a interface gráfica responsiva enquanto se aguardam dados do módulo embarcado. A solução envolveu o uso de *threads* ou o agendamento de tarefas (`root.after()` no Tkinter) para leitura serial, evitando travamentos da UI.
  • Design de Interface para Iniciantes: Criar uma UI que fosse intuitiva e não intimidadora para usuários sem experiência em programação. Isso envolveu simplificar os controles e fornecer feedback visual claro.
  • Tratamento de Erros de Comunicação: Implementar mecanismos robustos para lidar com desconexões do hardware, erros de leitura serial e outras falhas de comunicação.

A superação desses desafios foi fundamental para o desenvolvimento de uma ferramenta educacional estável e acessível, demonstrando proficiência em **Desenvolvimento de Sistemas Multiplataformas**.
